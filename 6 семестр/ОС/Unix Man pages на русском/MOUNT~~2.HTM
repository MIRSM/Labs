MOUNT(8)

НАЗВАНИЕ
mount - монтирование файловой конструкции

СИНТАКСИС 

 /etc/mount [special-device directory [-r]]

ОПИСАНИЕ
mount сообщает системе, что переносимая файловая конструкция присутствует на устройстве special-device. Эта файловая конструкция монтируется на основе каталога directory. Этот каталог должен уже присутствовать, имя directory становится корневым именем монтируемой файловой конструкции. Каталог directory должен быть пустым. Если в нем имеются файлы, то они будут как бы удалены при монтировании этого каталога и появятся снова, когда он будет демонтирован.

Команды mount и unmount поддерживают таблицу смонтированных устройств. Если какое-либо специальное устройство вызывается без всяких аргументов, то mount высвечивает имя этого устройства, имя каталога смонтированной файловой конструкции и информацию о том, предназначена ли данная файловая конструкция только для чтения, а также дату монтировки.

Последний необязательный аргумент указывает на то, будет ли монтируемый файл предназначенным только для чтения. Физически защищенные от записи файловые конструкции должны монтироваться только этим способом, иначе при модификации времени доступа будут возникать ошибки независимо от того, имеются ли попытки явного чтения или нет.

unmount удаляет переносимые файловые конструкции, смонтированные на устройстве special-device.

ФАЙЛЫ 

 /etc/mntab           Таблица смонтированных устройств.
 /etc/default/filesys Данные файловой системы.

СМ. ТАКЖЕ
unmount(8), mnt(1), mount(2), mnttab(3), default(3)

ДИАГНОСТИКА
mount выдает предупреждение, если подлежащая монтировке файловая конструкция в текущий момент смонтирована под другим именем.

Занятые файловые конструкции не могут быть демонтированы с помощью unmount. Файловая конструкция считается занятой, если в ней содержится открытый файл или рабочий каталог какого-либо пользователя.

ПРИМЕЧАНИЯ
Только привилегированный пользователь может использовать команду mount.

Файловые конструкции тестируются в некоторой степени, и в общем случае - неразумно монтировать испорченные файловые конструкции.

Насторожитесь, если при нахождении в однопользовательском режиме команды, которые ищут в /etc/mnttab аргументы, используемые по умолчанию (например, df, ncheck, qout, mount и unmount) дают неправильные результаты (вследствие порчи /etc/mnttab из-за остановки системы без команды shutdown) или вообще не дают результатов (из-за пустого /etc/mnttab после выполнения shutdown).

В многопользовательском режиме это не составляет проблемы; /etc/rc инициализирует /etc/mnttab так, что тот содержит только /dev/root, и последующие монтировки модифицируют его соответствующим образом.

Команды mount(8) и unmount(8) используют файл блокировки для обеспечения исключительного доступа к /etc/mnttab. Команды, которые просто читают его (они упомянуты выше) не делают этого, и поэтому может случиться, что они попадают в испорченное окно. На практике это не составляет проблемы, так как операции mount и unmount не слишком часто используются.

При монтировании файловой системы на гибком диске вам не нужно использовать каждый раз один и тот же каталог directory. Однако, если вы поступаете именно так, полные имена файлов согласуются в каждом случае.

Дискеты не должны быть защищенными от записи с помощью наклейки при выполнении монтирования на них файловых систем. Перед тем, как вынуть дискету из дисковода, всегда демонтируйте на ней файловую систему с помощью unmount. В противном случае при последующем монтировании на этом диске понадобится запуск команды fsck.
